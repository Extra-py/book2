# EduStories

Интерактивная образовательная карта чтения: приветственный свиток, выбор класса, книги, одиночные и совместные задания, опыт и рейтинг.

## Самый простой способ опубликовать на GitHub Pages

Готовая версия сайта находится в папке `docs`. Никакая сборка на GitHub не требуется.

1. Загрузите **всё содержимое проекта** в корень репозитория GitHub. Папка `docs` обязательно должна попасть в репозиторий.
2. Откройте репозиторий на GitHub и перейдите в **Settings → Pages**.
3. В разделе **Build and deployment** выберите:
   - **Source:** `Deploy from a branch`;
   - **Branch:** `main`;
   - **Folder:** `/docs`.
4. Нажмите **Save**.
5. Подождите 1–5 минут и обновите страницу **Settings → Pages**. Там появится ссылка на сайт.

Важно: файлы `docs/index.html` и `docs/assets/...` должны отображаться на GitHub. Если в репозитории находится только папка `Книгочей`, а уже внутри неё лежит `docs`, GitHub Pages сайт не найдёт. Содержимое `Книгочей` нужно загрузить непосредственно в корень репозитория.

## Автоматическая публикация

В проекте также оставлен workflow `.github/workflows/deploy-pages.yml`. Если в **Settings → Pages** выбрать `GitHub Actions`, GitHub соберёт и опубликует проект автоматически после отправки изменений в `main` или `master`.

Используйте только один вариант источника публикации: либо `main /docs`, либо `GitHub Actions`.

## Локальный запуск

```bash
pnpm install
pnpm dev
```

Повторная сборка папки `docs`:

```bash
pnpm build
```
